Conception responsable : The Fork

TheFork est une plateforme de réservation de restaurants en ligne.
Elle permet aux utilisateurs de rechercher un établissement, consulter des avis, voir les menus, et réserver une table en quelques clics.

Utilité principale : Simplifier et optimiser le processus de réservation pour les clients et les restaurateurs.

Les ODD

Que sont les ODD ?

ODD signifie "Objectifs de Développement Durable". Ils couvrent l'intégralité des enjeux de développement dans tous les pays tels que le climat, la biodiversité, l’énergie, l’eau, la pauvreté, l’égalité des genres, la prospérité économique ou encore la paix, l’agriculture, l’éducation, etc.

ODD associés à Thefork

ODD 9
Mettre en place une infrastructure résiliente, promouvoir une industrialisation durable qui profite à tous et encourager l'innovation

Service numérique innovant pour un secteur traditionnel.

ODD 12
Établir des modes de consommation et de production durables

Meilleure gestion des réservations, permettant de réduire le gaspillage.

Conception responsable

Audit via l'analyse GreenIT

Eco Index : C
Bonnes pratiques
  • Les headers de cache control sont présents
  • Le site utilise moins de six domaines
  • Aucune image inutile téléchargée
  • Le nombre de requêtes HTTP sont limitées
  • Le nombre de fichiers css est limité
  • Une police de caractère standard est utilisée
  • Il n'y a aucun cookie surchargé
Mauvaises pratiques
  • En moyenne 1.72g de co² par visite
  • Des images sont retaillées dans le navigateur
  • Pas de feuille print css
  • 433 lignes de css et js ont été trouvées dans le code source html et n'ont donc pas été externalisées

Audit manuel

Les outils Lighthouse et Ecoindex sont bloqués sur le site Thefork. Nous avons donc effectué un audit manuel du site en plus de l'analyse GreenIT.

Points forts relevés
  • Bonne accessibilité aux fonctionnalités
  • Interface claire et intuitive
  • Le service Thefork permet de réduire dans une certaine mesure le gaspillage via la planification
Points faibles relevés
  • Peu ou pas de descriptions ARIA
  • Poids des pages élevé (images non optimisées et en grande quantité)
  • Beaucoup d'images sont dans des formats plus lourds (ex: .png)
  • Pas de version légère pour les petites connexions internet ou mobile
  • Peu sobre (animations inutiles lors de certaines interactions)

Recommandations

  • Retailler les images coté serveur afin de réduire la taille des téléchargements
  • Utiliser le format webp pour les images
  • Optimiser les assets (css/js) afin de garder le strict minimum, et exporter l'intégralité de ce code
  • Améliorer l'accessibilité en utilisant de manière constante les descriptions ARIA
  • Alléger le javascript (actuellement d'une taille de 9600kB, il pourrait être réalistiquement réduit de moitié en rendant le site plus sobre et en supprimant du code inutile ou redondant)
  • Thefork pourrait davantage inciter à la consommation locale responsable
